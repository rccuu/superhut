# 工大盒子的信任与隐私说明

这份说明不是写给会看源码的人，而是写给想先弄清楚这几个问题的人：

- 这个 App 会连到哪里
- 会在本地存什么
- 为什么要这些权限
- 更新是从哪里来的

如果你只想先看结论，可以先看这 4 点：

1. 工大盒子的源码、版本发布和更新入口都公开在 GitHub。
2. 当前公开代码里，主要业务域名是学校系统、校园生活服务提供方，以及 GitHub Releases。
3. 登录信息会保存在你的设备上，用来维持登录状态；不会额外转发到作者自建服务器。
4. About 页面里可以直接看到仓库、上游仓库、更新入口和这份说明对应的摘要。

## 工大盒子会连接哪些服务

下面是当前公开代码中可见的主要业务域名：

| 域名 | 作用 |
| --- | --- |
| `github.com` | 检查 GitHub Releases、打开仓库和更新页面 |
| `mycas.hut.edu.cn` | 湖南工业大学统一认证登录 |
| `jwxtsj.hut.edu.cn` | 教务系统相关功能 |
| `portal.hut.edu.cn` | 校园门户相关接口 |
| `v8mobile.hut.edu.cn` | 水、电、洗浴等校园生活服务 |
| `authx-service.hut.edu.cn` | 电费等认证相关服务 |
| `i.ilife798.com` | 饮水设备相关服务 |

当前代码里没有单独写死作者自己的业务后端域名。你在 App 里使用成绩、课表、水电、饮水这些功能时，请求会直接发到这些学校或服务提供方域名。

## 本地会存哪些数据

为了让你不用每次都重新登录，App 会在设备本地保存一部分数据：

- 用户名、登录方式
- 登录态 token / session
- 课表缓存、小组件数据
- 个人资料摘要，例如姓名、学院、班级、绩点摘要

当前实现里：

- 密码优先存放在系统安全存储中
- 部分登录态信息仍保存在应用本地存储中，用来维持登录状态

这些数据默认只保存在你的设备上，不会同步到作者私有服务器。

## 为什么需要这些权限

### 网络权限

用于访问学校系统、校园生活服务接口，以及读取 GitHub Releases 更新信息。

### 相机权限

用于扫码导入课表或扫描饮水设备二维码。不授予相机权限时，这些扫码功能不可用，但不会影响其他不相关功能。

### 位置权限

仅在部分校园生活服务 WebView 页面需要定位能力时请求。如果你不使用这类功能，可以不授予。

### 开机启动 / 唤醒 / 网络状态权限

Android 小组件和部分刷新场景需要这些权限来保持桌面组件可更新、可响应。

## 更新是怎么来的

工大盒子当前的版本检查直接读取 GitHub Releases 信息，应用内点击“检查更新”后，会跳转到 GitHub Release 页面查看说明并下载安装。

也就是说，更新来源不是隐藏服务器，而是公开仓库里的 Release。

## 你可以自己怎么检查

如果你还是不放心，可以直接做这几件事：

1. 在应用的 **关于工大盒子** 页面里打开当前仓库和版本发布页。
2. 对照仓库里的代码和 `scripts/build_android_release.sh`、`scripts/build_ios_quick.sh` 两个构建脚本。
3. 只从仓库公开的 Release 页面下载安装包。
4. 遇到看不懂的权限或联网行为，直接提 Issue，让维护者公开解释。

## 当前边界

- 工大盒子是第三方校园工具，不是学校官方应用。
- 这份说明的目标是把现有实现说清楚，不代表项目已经完成完整的专业安全审计。
- 后续版本会继续优先做两件事：把权限和兼容配置讲清楚，以及让发布产物更容易验证。
